## 1. Root cause — why manual testing appeared broken

Two independent problems masked each other:

### 1.1 Dual-daemon Discord gateway contention
A daemon was running on the **local Mac** AND on the **VPS**, both connecting to Discord with the **same bot token** (`iFleet`, app `1503479349683159220`). Discord routes each interaction to only one gateway session; the two fought. The operator was fixing/testing the local daemon while Discord delivered `/ship` commands to the VPS (old code). The local Mac daemon was **stopped** (`pm2 stop ifleet` locally) — the VPS is now the sole gateway holder.

Note: there is also a **second, unrelated Discord bot** in `#ifleet` — `ifleet` lowercase, app `1504853883888074804` — not from this codebase. It posts fake `PR-x`/`PR-recovered` completions and budget-pause spam. Unidentified; see §8.6.

See §2.

## 4. Deployment & the credential saga

The VPS daemon's `gh` (for `gh pr create`) authenticates via `GITHUB_TOKEN`. Sequence of what was wrong and how it was fixed:

1. Token was a **fine-grained PAT** (`github_pat_11B743…`) sourced from the VPS `/etc/environment`. Fine-grained PATs need **per-org approval** for org-owned repos; `weautomatehq1` had never granted it → the token saw `weautomatehq1/IFleet` as read-only → `Resource not accessible by personal access token (createPullRequest)`.
2. Fine-grained route is a dead end without multi-step org setup.
3. **Fix:** a fresh **classic PAT** (`ghp_…`, scopes `repo` + `read:org`) was created and added to **`/opt/ifleet/.env`** as `GITHUB_TOKEN`. `ecosystem.config.cjs` reads `.env` and merges it into each app's PM2 `env:` block, which overrides `/etc/environment`. Daemon restarted; confirmed runtime token prefix `ghp_`.

`read:org` was added after the first success because `gh pr edit --add-reviewer` needs it to resolve the reviewer login (without it, reviewer assignment failed *non-fatally* — F5 working as designed).

## 7. Hard rules / gotchas

- **One daemon per Discord token.** Never run a local daemon while the VPS daemon runs — they fight over the gateway. The VPS is the live environment; test there.
- **Production path is `/opt/ifleet`**, not `/root/IFleet`. The `fleet:*` scripts in `package.json` point at the wrong path (§8.2).
- **`GITHUB_TOKEN`** lives in `/opt/ifleet/.env`; `ecosystem.config.cjs` merges `.env` over `/etc/environment`. Must be a **classic** PAT (`repo` + `read:org`) — fine-grained PATs hit org-approval walls.
- **Deploy** = `git pull` on `/opt/ifleet` + `pm2 restart ifleet`. Use `pm2 restart ifleet` (not `pm2 restart ecosystem.config.cjs`, which also restarts/starts other apps).
- **SprintManager emits events; never calls GitHub directly** — all GitHub access flows through the queue bridge (CLAUDE.md load-bearing rule).
